Travel Minutes API

Alternative authentication and data storage for the web app Travel Minutes.

Public Endpoints*:

*Public endpoints do not require an Authorization header with the request.

POST, "/register"

  • Purpose: Register a new user.
  • Request body: {email: String, password: String}
  • Password must contain 7 to 12 characters.
  • Successful response: JSON Web Token

POST, "/login"

  • Purpose: User login.
  • Request body: {email: String, password: String}
  • Password must contain 7 to 12 characters.
  • Successful response: JSON Web Token

Private Endpoints*:

*Private endpoints require an Authorization header of type Bearer with a valid JSON Web Token.

POST, "/logout"

  • Purpose: User logout.
  • Must include Authorization header.
  • Successful response: "Logged out".

PATCH, "/change-password"

  • Purpose: Change a user's password.
  • Request body: {oldPassword: String, newPassword: String}
  • Password must contain 7 to 12 characters.
  • Must include Authorization header.
  • Successful response: "Password changed".

GET, "/all-entries"

  • Purpose: Fetch all entries of a given user.
  • Must include Authorization header.
  • Successful response: Array of entry objects.

GET, "/entry-id"

  • Purpose: Fetch a single entry by its id.
  • Request body: {entryId: String}
  • Must include Authorization header.
  • Successful response: Entry object.

POST, "/add-entry"

  • Purpose: Add an entry.
  • Request body: {date: String, place: String , seen: String, done: String, met: String, (OPTIONAL) label: String}
  • Max length: date = 10 characters; place, label = 50 characters each; seen, done, met = 100 characters each.
  • Must include Authorization header.
  • Successful response: Entry object.

DELETE, "/delete-entry"

  • Purpose: Delete an entry
  • Request body: {entryId: String}
  • Must include Authorization header.
  • Successful response: Entry object.

PATCH, "/edit-entry"

  • Purpose: Update an entry.
  • Request body: {entryId: String, seen: String, done: String, met: String }
  • Max length: seen, done, met = 100 characters each.
  • Must include Authorization header.
  • Successful response: Entry object.
